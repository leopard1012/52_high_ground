# 전체 구조도(설계)

## 1. 프로젝트 개요

`Ground`는 React + Vite 기반의 프론트엔드 앱입니다. 주요 목적은 기도 커뮤니티 데이터를 Firebase로 저장하고 실시간으로 동기화하는 것입니다.

## 2. 소스별 기능 설명

### 2.1 `src/index.jsx`

- React 앱 엔트리 포인트
- `App` 컴포넌트를 렌더링
- 전역 스타일 및 기본 라우팅이 여기에 연결됩니다

### 2.2 `src/App.jsx`

- 앱의 최상위 레이아웃
- 탭 네비게이션과 화면 전환 역할
- 인증 상태 및 전역 상태가 이 계층에서 관리됩니다

### 2.3 `src/components/*`

- `HomeTab.jsx`: 기도 카드 목록과 기도하기, 카드 등록 기능
- `ChallengeTab.jsx`: 챌린지 목록, 참여 진행도, 고민 상담 섹션
- `CrewTab.jsx`: 크루 랭킹, 크루 정보, 메시징/채팅 인터페이스
- `MeetingTab.jsx`: 기도 모임 일정 보기, 참석 관리
- `MyTab.jsx`: 사용자 프로필, 보유 배지, 씨앗 및 레벨 정보
- `LoginPage.jsx`: 로그인/회원가입 화면 및 OAuth 처리
- `FloatAnim.jsx`: UI 애니메이션 컴포넌트

### 2.4 `src/hooks/*`

- `useAuth.js`: Firebase 인증 상태 구독 및 사용자 상태 관리
- `useCards.js`: 카드 컬렉션 실시간 구독, CRUD 훅
- `useChallenges.js`: 챌린지/미션 상태 구독 및 진행도 관리
- `useConcerns.js`: 고민 상의 및 답변 상태 구독

### 2.5 `src/firebase/*`

- `config.js`: Firebase 앱 초기화, `auth`와 `firestore` 객체 생성
- `services/authService.js`: 인증 관련 API, 로그인/회원가입/구글 OAuth/로그아웃
- `services/cardService.js`: 카드 CRUD 로직, 기도하기 + `prayedBy` 업데이트
- 추가 파일 가능: `challengeService.js`, `concernService.js`, `crewService.js`, `meetingService.js`

### 2.6 `src/utils/ground-utils.js`

- 공통 유틸리티 함수
- 날짜 포맷, 레벨 계산, UI 데이터 변환 등

## 3. 소스별 관계도

```
index.jsx
  └ App.jsx
       ├─ LoginPage.jsx
       ├─ HomeTab.jsx
       │    └─ useCards.js -> cardService.js -> Firestore(cards)
       ├─ ChallengeTab.jsx
       │    └─ useChallenges.js -> challengeService.js -> Firestore(challenges)
       │    └─ useConcerns.js -> concernService.js -> Firestore(concerns)
       ├─ CrewTab.jsx
       │    └─ crewService.js -> Firestore(crews)
       ├─ MeetingTab.jsx
       │    └─ meetingService.js -> Firestore(meetings)
       └─ MyTab.jsx
            └─ useAuth.js -> authService.js -> Firestore(users)

firebase/config.js
  └ firebase services

utils/ground-utils.js
  └ 모든 컴포넌트에서 공통 계산, 포맷 등 사용
```

### 데이터 흐름 요약

- UI 컴포넌트는 `hooks`를 통해 Firebase 데이터를 구독
- `hooks`는 `services` 레이어의 CRUD 함수를 호출
- `services`는 Firestore API를 통해 데이터 읽기/쓰기
- 인증 상태는 `useAuth.js`에서 `onAuthStateChanged`로 처리

## 4. 데이터 구조

### 4.1 Firestore 컬렉션

#### `users`

- `userId`
  - `nickname`: string
  - `avatar`: string
  - `seeds`: number
  - `badges`: array<string>
  - `joinedDate`: timestamp
  - `currentTreeLevel`: number

#### `cards`

- `cardId`
  - `userId`: string
  - `user`: string
  - `avatar`: string
  - `text`: string
  - `emoji`: string
  - `bg`: string
  - `count`: number
  - `prayedBy`: array<string>
  - `createdAt`: timestamp
  - `updatedAt`: timestamp

#### `challenges`

- `challengeId`
  - `title`: string
  - `icon`: string
  - `totalDays`: number
  - `reward`: number
  - `createdAt`: timestamp
  - `participants`: map<string, { done: array<boolean> }>

#### `concerns`

- `concernId`
  - `userId`: string
  - `text`: string
  - `createdAt`: timestamp
  - `replies`: array<{
    userId: string,
    text: string,
    timestamp: timestamp
    }>
  - `status`: string

#### `crews`

- `crewId`
  - `name`: string
  - `seed`: number
  - `memberCount`: number
  - `members`: array<string>
  - `createdAt`: timestamp

#### `meetings`

- `meetingId`
  - `name`: string
  - `organizerId`: string
  - `date`: timestamp
  - `time`: string
  - `location`: string
  - `description`: string
  - `members`: array<string>
  - `maxMembers`: number
  - `attendances`: array<{
    userId: string,
    verified: boolean,
    timestamp: timestamp
    }>
  - `createdAt`: timestamp

### 4.2 프론트엔드 내부 상태

#### 카드 목록

- `cards`: array of card objects
- `loading`: boolean
- `error`: string|null

#### 챌린지 상태

- `challenges`: array of challenge objects
- `progressByUser`: array<boolean> 또는 객체 매핑

#### 사용자 프로필

- `user`: Firebase Auth 사용자 정보
- `profile`: Firestore `users/{userId}` 문서 데이터

## 5. 설계 방향 핵심

- **실시간 구독**: `onSnapshot()`으로 데이터 변경을 즉시 반영
- **인증 기반 접근**: Firebase Auth가 모든 주요 기능을 보호
- **서비스 분리**: UI, 훅, 서비스 레이어를 분리하여 유지보수성 향상
- **데이터 구조 통일**: Firestore 문서 구조를 중앙에서 표준화
- **확장성 고려**: `services` 및 `hooks` 확장으로 새로운 기능 추가 용이
