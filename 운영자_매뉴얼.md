# 운영자(개발자, 시스템관리자) 매뉴얼

## 1. 개요

이 문서는 `Ground` 프로젝트를 개발 및 운영하는 개발자와 시스템관리자용 매뉴얼입니다. 설치, 실행, 배포, 유지보수, Firebase 관리까지 포함합니다.

## 2. 개발 환경 설정

### 2.1 시스템 요구 사항

- Node.js 18 이상 권장
- npm 10 이상
- Windows / macOS / Linux 모두 지원
- Firebase 프로젝트 및 서비스 계정 정보

### 2.2 필수 설치

```bash
npm install
npm install firebase
```

### 2.3 환경 변수

- Firebase 웹 앱 설정 정보는 다음 항목을 포함해야 합니다.
  - `apiKey`
  - `authDomain`
  - `projectId`
  - `storageBucket`
  - `messagingSenderId`
  - `appId`

> `firebaseConfig`는 `src/firebase/config.js`에서 읽히도록 설정합니다. 환경 변수 파일(`.env`) 또는 별도 설정 파일을 사용하여 배포 환경을 분리합니다.

## 3. 프로젝트 구조 요약

```
src/
  App.jsx
  index.jsx
  components/
  firebase/
    config.js
    services/
  hooks/
  utils/
```

- `App.jsx`: 전체 앱 레이아웃과 탭 구조
- `components/`: 화면별 UI 컴포넌트
- `firebase/config.js`: Firebase 초기화
- `firebase/services/`: Firestore 및 인증 CRUD 로직
- `hooks/`: 컴포넌트별 데이터 구독 훅
- `utils/`: 공통 유틸리티

## 4. Firebase 구성 및 운영

### 4.1 Firebase 초기화

- Firebase 프로젝트 생성
- `firebaseConfig` 객체를 `src/firebase/config.js`에 등록
- Authentication, Firestore, Hosting 필요 시 활성화

### 4.2 인증 설정

- Email/Password 로그인
- Google OAuth 로그인
- 인증 후 첫 로그인 시 `users/{userId}` 문서 자동 생성
  - 기본 필드: `nickname`, `avatar`, `seeds`, `badges`, `joinedDate`, `currentTreeLevel`

### 4.3 Firestore 보안 규칙 예시

```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;
    }

    match /cards/{cardId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update, delete: if resource.data.userId == request.auth.uid;
    }

    match /challenges/{challengeId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }

    match /concerns/{concernId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update: if request.auth.uid == resource.data.userId;
    }

    match /crews/{crewId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }

    match /meetings/{meetingId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }
  }
}
```

### 4.4 Firestore 인덱스

- `cards.createdAt`
- `challenges.createdAt`
- `meetings.date`
- `users.joinedDate`

필요 시 Firebase 콘솔에서 인덱스를 추가합니다.

## 5. 개발 및 테스트

### 5.1 로컬 실행

```bash
npm run dev
```

### 5.2 빌드

```bash
npm run build
```

### 5.3 코드 수정 시 권장 절차

1. `components/*`에서 UI 수정
2. `hooks/*`에서 데이터 구독/상태 변경 로직 수정
3. `firebase/services/*`에서 Firestore CRUD 로직 검증
4. `firebase/config.js`로 Firebase 초기 설정 확인
5. `utils/ground-utils.js`에 공통 함수 추가

### 5.4 디버깅

- 네트워크: 브라우저 개발자 도구의 Network 탭
- Firestore 쿼리: Firebase 콘솔에서 읽기/쓰기 로그 확인
- 인증 오류: Firebase Auth 로그 확인
- 실시간 업데이트: `onSnapshot()` 콜백이 해제되지 않았는지 점검

## 6. 배포

### 6.1 정적 호스팅

- Vite 빌드 결과 `dist/` 폴더 생성
- Firebase Hosting 또는 다른 정적 사이트 서비스에 배포

- Firestore 사용량 확인: 읽기/쓰기 비용
- 인증 사용자 수 확인
